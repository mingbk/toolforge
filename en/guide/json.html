📋 JSON: The Complete Guide

From syntax to real-world use — master the universal data format

What Is JSON?

JSON stands for JavaScript Object Notation. It's a lightweight, plain-text data interchange format. It represents structured data in a human-readable way and is the most universal data format in modern web development.

Here's a JSON snippet describing a user: 

{
  "name": "Alice",
  "age": 28,
  "email": "alice@example.com",
  "tags": ["frontend", "React"]
}

This format is friendly to both humans and machines — you can read it directly, and programs can parse it instantly.

TL;DR: JSON is a "key-value + arrays" text format for data. Every programming language can read and write it — it's the lingua franca of data exchange.

Why Format and Validate JSON?

1. Minified JSON Is Unreadable

APIs often return minified (compact) JSON to save bandwidth: 

{"code":200,"data":{"users":[{"id":1,"name":"Alice"}]}}

After formatting, it becomes instantly clear: 

{
  "code": 200,
  "data": {
    "users": [
      { "id": 1, "name": "Alice" }
    ]
  }
}

Formatting = indentation + line breaks + syntax highlighting. You can see the data structure at a glance.

2. One Trailing Comma Can Break Everything

JSON syntax is strict. A single mistake causes parse failure. Common errors:

  • ❌ Trailing comma after last element: {"a": 1, "b": 2,}
  • ❌ Unquoted keys: {name: "Alice"}
  • ❌ Single quotes instead of double: {'name': 'Alice'}
  • ❌ Missing quotes or mismatched brackets

A validator tells you exactly what's wrong in real time, so you catch errors before deploying.

Common JSON Use Cases

🔌 API Debugging

Nearly every REST API today uses JSON. When you call an endpoint with curl or Postman, you get JSON back. The first thing you do with a wall of minified JSON is format it — otherwise you'll never figure out which nesting level holds data.users[5].address.city.

⚙️ Configuration Files

package.json, VSCode's settings.json, ESLint's .eslintrc.json — JSON is the default config format in the JavaScript ecosystem. A malformed config file can prevent your entire project from running.

📦 Data Exchange

In a decoupled frontend-backend architecture, the frontend sends JSON to the backend and the backend returns JSON. When different languages (Java ↔ Python ↔ Go) need to share data, serializing to JSON strings is the standard approach.

🗄️ Databases

PostgreSQL has a native JSONB column type, MongoDB documents are essentially JSON (BSON), and Redis's JSON module makes JSON a standard cache format.

Format vs Minify: When to Use Which?

Format: During development, debugging, writing config files, sharing with colleagues — you need readability.
Minify: For production transport, database storage, URL parameters — you need to save bytes.

FAQ

Q: What's the difference between JSON and a JavaScript object?

JSON is a plain-text format; a JavaScript object is an in-memory data structure. JSON keys must be double-quoted, and you can't include functions, undefined, or NaN. Use JSON.parse() to convert a JSON string into a JavaScript object.

Q: Why do I get "Unexpected token" when parsing JSON?

99% of the time it's: a trailing comma, single quotes, unquoted keys, or invisible characters from copy-paste. Paste your content into a formatter — it will highlight the error location.

Q: Does JSON support Chinese characters?

Yes, fully. JSON string values can contain any Unicode character — Chinese, Japanese, emoji. No escaping needed: "message": "Hello world 🌍".

Q: How deep can JSON nest?

No theoretical limit, but browsers and parsers usually enforce a default nesting depth (typically 100–1000 levels). In practice, if your data is nested deeper than 5 levels, it's time to refactor.
