🔑 RSA 非对称加密：两把钥匙的魔法

理解公钥/私钥机制、RSA-OAEP 填充方式，以及如何安全地生成和使用 RSA 密钥

什么是非对称加密？

非对称加密最大的特点是有两把不同的钥匙：一把公钥（Public Key），一把私钥（Private Key）。公钥可以公开给任何人，用来加密数据；私钥必须严格保密，用来解密数据。

这和 AES 完全不同——AES 加解密用同一把钥匙，叫"对称加密"。非对称加密解决的是"密钥分发"问题：你怎么把密钥安全地传给对方？有了 RSA，你只需要拿到对方的公钥就能加密，而私钥永远不需要离开对方的设备。

类比：公钥就像一个公开的邮箱地址——任何人都可以把信投进去。私钥就像邮箱的钥匙——只有持有钥匙的人才能打开邮箱读取信件。

RSA 的数学原理（简化版）

RSA 的安全性建立在大数质因数分解的困难性上。简单来说：

  • 随机选两个非常大的质数 p 和 q（比如各 1024 位）
  • 计算 n = p × q。n 就是"模数"，出现在公钥和私钥中
  • 选择一个公钥指数 e（通常是 65537）
  • 计算私钥指数 d，使得 (m^e)^d ≡ m (mod n)

加密就是计算 c = m^e mod n，解密就是 m = c^d mod n。核心难题：如果你只知道 n（公开的），要算出 p 和 q 需要分解一个 2048 位的超大整数——以目前的计算能力，暴力分解需要的时间远超宇宙年龄。

RSA-OAEP：为什么不能直接用 RSA？

教科书上的"RSA 加密 = m^e mod n"只是基础，直接用有严重安全问题。实际使用中必须加上填充方案（Padding Scheme）。OAEP（Optimal Asymmetric Encryption Padding，最优非对称加密填充）是目前的标准方案。

填充方案安全性推荐？
无填充（Textbook RSA）❌ 多项攻击绝不使用
PKCS#1 v1.5⚠️ 有已知弱点避免使用
OAEP (SHA-256)✅ 安全✅ 推荐

OAEP 在加密前引入了随机填充，确保同样的明文每次加密产生不同的密文，同时防止了多种数学攻击。浏览器 Web Crypto API 直接支持 RSA-OAEP： 

// 使用 Web Crypto API 进行 RSA-OAEP 加密
const pubKey = await crypto.subtle.importKey(
  'spki', pemToArrayBuffer(pem),
  { name: 'RSA-OAEP', hash: 'SHA-256' },
  false, ['encrypt']
);
const ciphertext = await crypto.subtle.encrypt(
  { name: 'RSA-OAEP' }, pubKey, new TextEncoder().encode(plaintext)
);

一般建议：2048 位足够应对绝大多数场景。如果你需要更长期的安全保障（比如加密需要保存 10 年以上的数据），或者你的安全策略要求 4096 位，就选 4096 位。现代浏览器生成 2048 位密钥只需要 1-3 秒，4096 位则需要 10-30 秒。

RSA 的长度限制和解决方案

RSA 一个核心限制是：加密的明文长度不能超过密钥长度减去填充的长度。2048 位 RSA-OAEP（SHA-256）最多加密约 190 字节。如果你想加密一张图片或一段长文本，RSA 直接做不到。

标准做法——混合加密（Hybrid Encryption）：

  1. 生成一个随机的 AES 密钥
  2. 用 AES 加密实际数据（速度快，无长度限制）
  3. 用 RSA 加密那个 AES 密钥（AES 密钥只有 32 字节，RSA 轻松加密）
  4. 把 RSA 加密的密钥和 AES 加密的数据一起发给对方

这就是 HTTPS、PGP、SSH 等几乎所有现代加密协议的做法。RSA 负责安全地传递密钥，AES 负责高效地加密数据——两者互补。

实际场景：什么时候用 RSA？

  • 🔐 安全密钥交换：在不安全的信道上协商出一个共享密钥（TLS 握手）
  • ✍️ 数字签名：用私钥签名，任何人用公钥验证——证明消息确实来自你
  • 📧 加密邮件：用收件人的公钥加密邮件内容
  • 🔑 SSH 登录：你本地生成密钥对，把公钥放到服务器，登录时用私钥认证
  • 🔄 API 鉴权：用私钥对请求签名，服务端用公钥验证——JWT 的 RS256 算法

常见问题

Q: 公钥能反推出私钥吗？

不能。从公钥（包含模数 n 和指数 e）反推私钥（指数 d），需要知道 n 的质因数 p 和 q。分解一个 2048 位的巨大整数，即使动员全世界的算力也需要远超宇宙年龄的时间。但注意：量子计算机（Shor 算法）理论上可以在多项式时间内分解大整数，因此 RSA 不是"后量子安全"的。

Q: 为什么公钥指数 e 通常是 65537？

65537（二进制 10000000000000001）是一个质数，且只有两个 1 位。这使得加密运算（m^e mod n）可以通过"平方-乘"快速算法极快地完成。选择 65537 而不是 3（更小的选择）是为了防止低指数攻击。65537 是性能和安全性之间的最优平衡点。

Q: RSA 密钥应该多久换一次？

取决于使用场景。NIST 建议 RSA 2048 位密钥的安全使用期限到 2030 年。对于高安全性场景，建议每年轮换；对于一般应用，2-3 年轮换一次是合理的。关键是：一旦怀疑私钥泄露，立刻更换——RSA 密钥没有"前向安全性"，泄露的私钥能解密所有历史密文。
