什么是 JSON？

JSON 全称 JavaScript Object Notation（JavaScript 对象表示法），是一种轻量级的纯文本数据交换格式。它用人类可读的文本存储和传输结构化数据，是当今 Web 开发中最通用的数据格式。

比如下面这段 JSON，描述了一个用户的信息： 

{
  "name": "张三",
  "age": 28,
  "email": "zhangsan@example.com",
  "tags": ["前端", "React"]
}

这种格式对人和机器都非常友好——你可以直接读，程序也能轻松解析。

一句话总结：JSON 就是把数据写成一种"键值对 + 数组"的文本格式，任何编程语言都能读写它，是系统间交换数据的"通用语言"。

为什么需要格式化和校验 JSON？

1. 压缩后的 JSON 根本没法读

API 返回的 JSON 为了节省带宽往往是一行到底的"压缩"格式： 

{"code":200,"data":{"users":[{"id":1,"name":"张三"}]}}

格式化后立刻清晰： 

{
  "code": 200,
  "data": {
    "users": [
      { "id": 1, "name": "张三" }
    ]
  }
}

格式化 = 缩进 + 换行 + 语法高亮，一眼就能看清数据结构。

2. 一个逗号就能炸掉整个系统

JSON 的语法非常严格，一个小错误就会导致解析失败。常见错误包括：

  • ❌ 最后一个元素后面多了逗号：{"a": 1, "b": 2,}
  • ❌ 键名没有用双引号：{name: "张三"}
  • ❌ 用了单引号：{'name': '张三'}
  • ❌ 缺少引号或括号不匹配

JSON 语法规则速查

规则说明示例
数据格式键值对，键必须用双引号"name": "张三"
字符串必须用双引号（不能用单引号）"hello"
数字整数或浮点数，不加引号42、3.14、-1
布尔值小写，不加引号true、false
空值小写 nullnull
数组方括号，元素逗号分隔[1, 2, 3]
对象花括号，键值对逗号分隔{"a": 1, "b": 2}
嵌套对象和数组可以任意嵌套{"items": [{"id": 1}]}
注释❌ JSON 不支持注释需要注释用 JSONC 或 YAML

JSON 常见使用场景

🔌 API 调试

现在几乎所有的 REST API 都用 JSON 传输数据。你用 curl 或 Postman 调接口，返回的就是 JSON。看到一大坨压缩 JSON 的时候，格式化是第一件事——不然你根本不知道 data.users[5].address.city 在哪一层。

⚙️ 配置文件

前端项目的 package.json、VSCode 的 settings.json、ESLint 的 .eslintrc.json……JSON 是 JavaScript 生态的默认配置文件格式。一个格式错误的配置文件会让整个项目跑不起来。

📦 数据交换

前后端分离架构下，前端发 JSON 给后端，后端返回 JSON 给前端。不同语言之间（Java ↔ Python ↔ Go）交换数据，序列化成 JSON 字符串是最标准的做法。

🗄️ 数据库

PostgreSQL 有原生 JSONB 列类型，MongoDB 的文档本身就是 JSON（BSON），Redis 的 JSON 模块也让 JSON 成了缓存的标准格式。

格式化 vs 压缩：什么时候用哪个？

格式化：开发调试、写配置文件、分享给同事看的时候——你需要可读性。
压缩：生产环境传输、存储到数据库、放到 URL 参数里的时候——你需要节省字节。

常见问题

Q: JSON 和 JavaScript 对象有什么区别？

JSON 是纯文本格式，JavaScript 对象是内存中的数据结构。JSON 的键必须用双引号，不能用函数、undefined、NaN 等 JS 特有的类型。用 JSON.parse() 可以把 JSON 字符串转成 JavaScript 对象。

Q: 为什么我的 JSON 解析报 "Unexpected token"？

99% 的情况是：最后多了逗号、用了单引号、键名没加双引号、或者复制粘贴时带进了不可见字符。把内容贴到格式化工具里，它会高亮错误位置。

Q: JSON 支持中文吗？

完全支持。JSON 的字符串值可以包含任意 Unicode 字符，包括中文、日文、emoji。不需要转义，直接写就行："message": "你好世界 🌍"。

Q: JSON 可以有多深？

理论上没有限制，但浏览器和解析库通常有默认的嵌套深度限制（一般是 100-1000 层）。实际开发中，超过 5 层的深层嵌套就说明数据结构该重构了。
