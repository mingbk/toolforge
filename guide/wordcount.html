📝 字数统计全攻略：中文、英文、混排到底怎么算？

别以为"数数"很简单——中英文的字数统计逻辑完全不同

字符数、字数、单词数——先搞清楚这三个概念

很多人在用字数统计工具的时候，分不清"字符"和"字"的区别。这在纯英文环境里问题不大，但在中文环境里——完全不是一回事。

  • 字符数（Character Count）：所有看得见的符号都算一个字符。字母、汉字、标点、空格、换行符——统统算进去。"Hello 你好" 的字符数是 8（H-e-l-l-o-空格-你-好）。
  • 字数（Word Count）：这个最复杂。英文按空格分词，"Hello world" 是 2 个词。但中文没有空格——一个汉字通常就是一个"字"。
  • 字符数（无空格）：去掉所有空白字符后的字符数，用于需要精确计数的场景（比如短信计费）。
核心区别：英文统计"单词"（words），中文统计"字"（characters as words）。"I love coding" = 3个单词；"我爱编程" = 4个字。

CJK文字 vs 拉丁文字：两种完全不同的计数逻辑

CJK（中日韩）文字属于表意文字，每个字符本身就代表一个完整的意思单位。拉丁文字（英语、法语、西班牙语等）属于拼音文字，需要通过空格把字母串成单词才能表达完整意思。

CJK计数规则

中文、日文、韩文不存在"单词"的概念（韩文虽然有空格，但统计上通常也按字符数处理）。一个汉字、一个假名、一个谚文字符，在计数时各算一个字。 

// 中文：每个汉字算一个字
"今天天气真好" → 中文字数：6

// 日文：假名和汉字都算
"今日は天気がいい" → 字符数：9

拉丁文字计数规则

英文等语言通过空格分词。但也有一堆边界情况：

  • 连字符："state-of-the-art" 算 1 个词还是 4 个？大多数工具算 1 个。
  • 缩写："I.T. support" —— 这个点让分词变得复杂。
  • 撇号："don't" 算 1 个词，"it's" 算 1 个词。
  • 数字："1,000 dollars" —— 1,000 算 1 个词。

混合文本的统计陷阱

真实的文本往往是中英混排的。比如： 

"我们使用 Python 和 Rust 来构建 API 服务"

这里面有中文（"我们使用""来构建""服务"），有英文单词（"Python""Rust"），还有缩写（"API"）。如果纯按空格分词，中文部分会连成一大坨；如果纯按字符数，英文单词又会被拆散。

Word Count 工具的混合策略：先用正则提取所有中文字符单独计数（[\u4e00-\u9fff] 范围），再用空格分词统计英文单词，过滤掉纯中文段。这样 "我愛 Python 编程" 会被统计为：中文字数 4 + 英文单词数 1。

实用建议：如果你要投稿、写论文、做SEO内容——中文字数和英文单词数分开看。大多数平台的"字数要求"指的是中文字数，英文部分按单词数折算（通常 1 个英文单词 ≈ 2 个中文字符）。

行数和段落数：不只是换行那么简单

行数按换行符 \n 分割，段落数按空行（两个连续的换行符）分割。 

这是第一段的内容。
这里还是第一段。

这是第二段。
中间隔了一个空行。

上面这段文本：4 行，2 个段落。

注意：段落分割用的是"至少一个空行"规则。如果你用 <br> 换行（单换行而非双换行），在纯文本统计中不会被识别为新段落。Markdown 里两个换行才等于一个段落分隔符。

常见问题

Q: 标点符号算不算字数？

标点算字符数，但不影响"字数"和"单词数"的统计。

Q: 空格算字符吗？

算。所以你能看到"总字符数"和"字符数（无空格）"两个指标——差值就是你文本里的空白字符数量。

Q: 中文和英文混排的字数怎么算？

分开统计：中文字数单独显示，英文单词数单独显示。你可以根据需要自己合计。

Q: 为什么我的中文文本"英文单词数"不为0？

如果你输入了空格、数字、英文标点——它们会被按空格分词后算作"单词"。这是预期行为：混合文本中，非纯中文段落的 token 都被计入了。
